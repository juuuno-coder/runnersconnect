# 화면 설계서 (Screen Design Document) - v0.1

본 문서는 **Runner's Connect** 플랫폼의 주요 화면 구성과 사용자 흐름을 정의합니다.

---

## 1. 공통 레이아웃 (Common Layout)

- **Header**: 로고, 대회 목록, 랭킹, 로그인/회원가입 (로그인 시: 마이페이지, 로그아웃).
- **Footer**: 회사 정보, 이용약관, 개인정보처리방침, 고객센터 연락처.

---

## 2. 주요 화면 상세

### 2.1. 메인 랜딩 및 대회 목록 (Main / Race List)

- **Hero Section**: 현재 가장 인기 있는 또는 곧 마감되는 대회 배너.
- **Race Cards**:
  - 대회 이미지, 대회명, 일시, 장소.
  - 접수 상태 태그 (접수중, 마감임박, 접수예정, 종료).
  - 종목 요약 (5K, 10K, Half 등).
- **검색/필터**: 지역별, 월별, 종목별 필터링.

### 2.2. 대회 상세 및 신청 (Race Detail & Registration)

- **상세 정보**: 대회 소개, 코스도, 기념품 상세 이미지, 시상 내역.
- **신청하기 버튼**: 클릭 시 신청 폼으로 이동.
- **신청 폼 (Step-by-step)**:
  1. **종목 선택**: 5K, 10K 등 (잔여 슬롯 실시간 표시).
  2. **참가자 정보 확인**: 이름, 생년월일, 성별 (회원정보 자동 로드).
  3. **참가 유형**: [개인 참가] / [단체 참가].
     - 단체 참가 시: '그룹 코드 입력' 필드 활성화.
  4. **옵션 선택**: 기념품 사이즈, 추가 옵션 (물품보관 등).
  5. **배송지 입력**: 기본 배송지 불러오기 또는 신규 입력.

### 2.3. 결제 및 완료 (Payment & Confirmation)

- **결제 페이지**: 주문 요약, 할인 적용(단체 할인 등), 포트원(아임포트) 결제 모듈 실행.
- **신청 완료**: 결제 성공 후 "등번호(Bib)" 확인, 카카오 알림톡 발송 안내.

### 2.4. 마이페이지 / 러닝 포트폴리오 (My Page / Portfolio)

- **대시보드**:
  - 상단: 나의 등급(엠블럼), 총 완주 횟수, 총 누적 거리.
  - 개인 기록(PB): 5K, 10K, Full 각각의 최고 기록 카드.
- **신청 내역/티켓**:
  - 예정된 대회 카드 -> 클릭 시 **현장 수령용 QR 코드** 팝업.
  - 주소지 변경 버튼 (마감 전까지 노출).
- **공인 기록 히스토리**:
  - 주최측 주관 대회의 완주 기록 리스트.
  - [완주증 보기/다운로드] 버튼.

### 2.5. 관리자 페이지 (Admin Portal)

- **대시보드**: 전체 매출, 누적 참가자, 진행 중인 대회 접수율 그래프.
- **대회 관리**: 대회 생성, 슬롯 설정, 주소지 변경 마감일 설정.
- **참가자 관리**: 신청 명단 조회, 환불 처리, 등번호 수동 배정.
- **기록 센터**: 기록 대행사 엑셀 업로드, 기록 오차 수정.
- **배송 센터**: 운송장 번호 대량 업로드, 배송 상태 업데이트.

---

## 3. 사용자 흐름 (User Flow)

1. **대회 탐색**: 메인 -> 대회 상세 확인.
2. **신청 시도**: 신청 버튼 클릭 -> (미로그인 시) 로그인/가입 이동 -> 신청 폼 입력.
3. **결제**: 포트원 결제 -> 성공 시 완료 페이지.
4. **대회 전**: 마이페이지에서 배송지 확인/변경.
5. **대회 당일**: 마이페이지 접속 -> QR 코드 제시 -> 기념품 수령.
6. **대회 후**: 기록 확정 알림 수신 -> 마이페이지에서 기록 확인 및 완주증 공유.
