# Running Race Platform Implementation Plan

이 문서는 **러닝 대회 플랫폼** 개발을 위한 초기 계획 및 기술 스택 제안을 담고 있습니다.

## 1. 프로젝트 개요

- **목표:** 대규모 참가자 등록 관리, 배번호(Bib) 및 굿즈 관리, 러닝 기록 저장 및 조회 기능을 제공하는 플랫폼 구축.
- **핵심 과제:** 등록 시작 시의 높은 트래픽 처리, 효율적인 참가자 데이터 관리, 사용자 친화적인 모바일 경험.

## 2. 기술 스택 제안

### 옵션 A: Next.js + Supabase (추천)

- **장점:**
  - **확장성:** 서버리스 아키텍처로 갑작스러운 트래픽 증가에 강함.
  - **실시간성:** Supabase의 Realtime 기능을 통한 실시간 기록 업데이트 가능.
  - **개발 속도:** `whymove` 프로젝트에서 이미 사용 중인 스택으로 익숙함.
- **구성:** Next.js (App Router), Tailwind CSS, Supabase (Auth, DB, Storage).

### 옵션 B: Ruby on Rails (사용자 고려 사항)

- **장점:**
  - **생산성:** "Convention over Configuration"으로 빠른 프로토타이핑 가능.
  - **안정성:** 성숙한 프레임워크로 복잡한 비즈니스 로직(결제, 환불 등) 처리에 유리.
- **구성:** Rails 7, Hotwire/Turbo (SPA 느낌의 UX), PostgreSQL.

## 3. 주요 기능 명세

### 1단계: 참가자 등록 및 관리

- **등록 폼:** 동적 폼 (티셔츠 사이즈, 기념품 선택, 비상 연락처 등).
- **결제 연동:** 토스페이먼츠 또는 포트원 연동.
- **배번호 자동 할당:** 결제 완료 순서에 따른 배번호 부여 로직.

### 2단계: 기록 관리 및 조회

- **기록 업로드:** 관리자용 CSV/Excel 대량 업로드 또는 API 연동.
- **개인 페이지:** 본인의 기록 차트 및 완주 인증서 다운로드.
- **실시간 리더보드:** 부문별(남/녀, 연령대 등) 순위 집계.

### 3단계: 운영 도구

- **체크인 관리:** QR 코드를 활용한 현장 배부 확인.
- **메시징:** 참가자 대상 안내 문자/알림톡 발송 모듈.

## 4. 다음 단계

1.  **기술 스택 결정:** 어떤 스택으로 시작할까요?
2.  **프로젝트 기초 설정:** 선택한 스택으로 프로젝트 초기화를 진행하겠습니다.
3.  **DB 스키마 설계:** 참가자 및 대회 정보 테이블 설계를 시작하겠습니다.
